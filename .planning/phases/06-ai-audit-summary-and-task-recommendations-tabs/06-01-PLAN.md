# Plan 1: AI Audit Summary and Task Recommendations Tabs

<objective>
Implement the backend and frontend for the AI Audit Summary and Task Recommendations. This includes adding new cached columns to the database, implementing server endpoints using Gemini 2.5 Flash, and adding a beautiful tabbed user interface (Findings | Summary | Tasks) to the dashboard.
</objective>

<task>
<read_first>
- db/schema.sql
- src/db.js
</read_first>
<action>
1. Modify `db/schema.sql` to add `ai_summary` (TEXT) and `ai_tasks` (TEXT) columns to the `audits` table.
2. Update the audits table creation query in `db/schema.sql` to:
```sql
CREATE TABLE IF NOT EXISTS audits (
  id INTEGER PRIMARY KEY,
  project_id INTEGER REFERENCES projects(id),
  url TEXT NOT NULL,
  started_at TEXT, finished_at TEXT,
  lighthouse_perf REAL, lighthouse_a11y REAL, lighthouse_seo REAL,
  status TEXT,  -- running | done | error
  progress TEXT,
  ai_summary TEXT,
  ai_tasks TEXT
);
```
3. Update `src/db.js` to run migration queries on existing databases so that starting the dashboard dynamically adds these columns if they don't exist:
```js
  const db = new Database(absoluteDbPath);
  
  const schemaPath = path.join(projectRoot, 'db', 'schema.sql');
  const schema = fs.readFileSync(schemaPath, 'utf8');
  
  db.exec(schema);

  // Dynamic migration for existing databases
  try {
    db.exec('ALTER TABLE audits ADD COLUMN ai_summary TEXT');
  } catch (e) {
    // Column already exists, ignore error
  }
  try {
    db.exec('ALTER TABLE audits ADD COLUMN ai_tasks TEXT');
  } catch (e) {
    // Column already exists, ignore error
  }
```
</action>
<acceptance_criteria>
- `db/schema.sql` contains `ai_summary TEXT` and `ai_tasks TEXT` in the `audits` table creation.
- `src/db.js` contains `ALTER TABLE audits ADD COLUMN ai_summary TEXT` and `ALTER TABLE audits ADD COLUMN ai_tasks TEXT` in try/catch blocks.
</acceptance_criteria>
</task>

<task>
<read_first>
- server.js
</read_first>
<action>
1. In `server.js`, add two endpoints: `POST /api/audits/:id/ai-summary` and `GET /api/audits/:id/summary`.
2. The `POST /api/audits/:id/ai-summary` endpoint should:
   - Check if GEMINI_API_KEY is configured in the environment.
   - Fetch the audit from the database.
   - Fetch all findings for the audit.
   - Summarize the findings to optimize token usage. Construct an aggregated counts breakdown by category and severity (critical/serious/moderate/minor), and a list of unique finding titles with their category and severity.
   - Prompt Gemini 2.5 Flash to generate:
     1. An "AI Audit Summary" in markdown format. It should include: an Overview, a Severity Breakdown, Key Highlights, and Recommendations.
     2. A "Prioritized Tasks List" in standard JSON list format (each task has: `priority` [1, 2, or 3], `title`, `description`, `category` [e.g. accessibility, performance, links, seo, console, network]).
     - *Important*: To ensure we get clean outputs, we can ask Gemini to return a single JSON object with `summary` (markdown text) and `tasks` (array of task objects) using JSON mode or structured outputs.
     - *SDK JSON mode syntax*: `response = await ai.models.generateContent({ model: 'gemini-2.5-flash', contents: prompt, config: { responseMimeType: 'application/json' } })`
   - Store the generated markdown `summary` and JSON stringified `tasks` in the `ai_summary` and `ai_tasks` columns of the `audits` table.
   - Return `{ summary, tasks }`.
3. The `GET /api/audits/:id/summary` endpoint should:
   - Fetch `ai_summary` and `ai_tasks` from the `audits` table.
   - Return `{ summary, tasks: JSON.parse(tasks) }` if cached, or return `{ summary: null, tasks: null }` if not cached.
</action>
<acceptance_criteria>
- `server.js` contains `/api/audits/:id/ai-summary` and `/api/audits/:id/summary` endpoints.
- `/api/audits/:id/ai-summary` saves generated data to SQLite `audits` table.
- `/api/audits/:id/summary` returns cached values if they exist in the DB.
</acceptance_criteria>
</task>

3. Listen to tab button clicks, switch active classes, show/hide `#findings-tab-content`, `#summary-tab-content`, and `#tasks-tab-content`.

   - `##` and `###` headers (replace with `<h2>` and `<h3>`).

   - Render tasks as cards with a checkbox, category badge, priority badge, title, and description.
   - Keep checkboxes interactive: store checked state in local storage (keyed by audit ID and task index) so that they persist when switching tabs.
7. Reset the tabs container and active tab to 'findings' when a new project or audit is selected, and show/hide the tabs container depending on whether an audit is selected.
