# Phase 3: MCP Server - Research

## MCP SDK Integration
The project requires building a Model Context Protocol (MCP) server. We must use the official `@modelcontextprotocol/sdk` package for Node.js.
The server needs three tools:
1. `run_audit(url: string)`: Spawns `node audit.js <url>`, waits for completion, and returns the basic audit result.
2. `get_findings(auditId: number, format: 'markdown' | 'json')`: Queries the database for findings associated with an audit run and returns them formatted.
3. `compare_screenshots(findingId: number)`: Spawns `node compare.js <findingId>`, waits for completion, and returns the diff pixel stats.

### Architecture
- We'll create `mcp/index.js` (or `.ts` if we use ts-node, but since the rest of the project is standard ES modules, we can stick to ES modules). Let's use `index.js`.
- The MCP server uses `StdioServerTransport`.
- As decided in `03-CONTEXT.md`, execution integration is via `child_process.spawn` or `exec` to invoke `node audit.js <url>` and `node compare.js <findingId>`. This shields the MCP stdout from the console noise in those scripts.
- As decided in `03-CONTEXT.md`, the MCP server's own internal diagnostics (`console.error` or `console.warn`) must be written to **stderr** only.

### Implementation Patterns
1. **Server Setup**:
```javascript
import { Server } from '@modelcontextprotocol/sdk/server/index.js';
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';
import { CallToolRequestSchema, ListToolsRequestSchema } from '@modelcontextprotocol/sdk/types.js';

const server = new Server({ name: 'site-audit', version: '1.0.0' }, { capabilities: { tools: {} } });
```

2. **Subprocess Spawning**:
```javascript
import { exec } from 'child_process';
import util from 'util';
const execPromise = util.promisify(exec);

async function runAudit(url) {
  const { stdout, stderr } = await execPromise(`node audit.js "${url}"`);
  // parse the JSON report generated by audit.js if needed, or just return success message
}
```
Wait, `audit.js` outputs a lot to stdout. So `exec` will capture it. That is fine, it just won't be piped to the MCP's stdout.

3. **Format Options**:
For `get_findings`, if format is 'markdown', we map over the SQLite results and produce a markdown table. If 'json', we just `JSON.stringify` them.
