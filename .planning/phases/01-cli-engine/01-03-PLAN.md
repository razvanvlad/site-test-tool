# Plan 3: Normalization & CLI Orchestrator

<objective>
Implement the normalizer to unify engine outputs into a standard format, and build `audit.js` to sequence the executions, insert into SQLite, and output the JSON report.
</objective>

<task>
<read_first>
- .planning/phases/01-cli-engine/01-CONTEXT.md
- db/schema.sql
</read_first>
<action>
Create `src/normalize.js` with functions to map raw findings to the standard database schema.
1. Export a single function `normalizeFindings(raw)` taking an object with `{ consoleErrors, failedRequests, axeViolations, lighthouseLhr, brokenLinks }`.
2. Map **Console Errors**: category `console`, severity `moderate`, source_tool `playwright`.
3. Map **Failed Requests**: category `network`, severity `moderate` or `serious` depending on status, source_tool `playwright`.
4. Map **Axe Violations**: category `accessibility`, severity maps from impact (`critical`, `serious`, `moderate`, `minor`), source_tool `axe`, extract selector from `nodes[0].target[0]`.
5. Map **Lighthouse**: Iterate over `lhr.audits`. For each where `score !== null && score < 1`, categorize based on `lhr.categories`. Severity `moderate` or `serious` based on score. source_tool `lighthouse`.
6. Map **Linkinator**: category `links`, severity `moderate` (if 403/999 set is_false_positive=1 or status='needs manual check' in notes), source_tool `linkinator`.
7. Ensure all returned objects match the `findings` table shape: `{ category, severity, title, description, selector, source_url, source_tool }`.
</action>
<acceptance_criteria>
- `src/normalize.js` exports `normalizeFindings`.
- Returns an array of objects matching the schema.
- Handles all 5 signal types.
</acceptance_criteria>
</task>

<task>
<read_first>
- src/engines/*.js
- src/db.js
- src/normalize.js
</read_first>
<action>
Create `audit.js` as the main CLI entry point.
1. Read URL from `process.argv[2]`. Handle missing URL gracefully with usage text.
2. Ensure `reports/screenshots/` exists using `fs.mkdirSync`.
3. Call `initDb()` to get the DB instance. Create an entry in `audits` with `status: 'running'`, `url`, and `started_at: new Date().toISOString()`. Get the inserted ID (`info.lastInsertRowid`).
4. Launch Playwright (`chromium.launch()`), create a page.
5. Generate screenshot path: `reports/screenshots/audit-${Date.now()}-full.png`.
6. Call `captureSignals` (Playwright).
7. Call `runAxe` using the same page.
8. Close Playwright browser.
9. Call `runLighthouse(url)`.
10. Call `checkLinks(url)`.
11. Pass all raw results to `normalizeFindings()`.
12. Iterate over normalized findings and `db.prepare('INSERT INTO findings...').run(...)` setting `audit_id` and `evidence_path` (screenshot path) where applicable.
13. Update `audits` row with `status: 'done'`, `finished_at`, and Lighthouse category scores.
14. Write JSON report: `fs.writeFileSync(reports/audit-${Date.now()}.json, JSON.stringify({ audit, findings }, null, 2))`.
15. Console log a summary showing total counts and breakdown by severity.
</action>
<acceptance_criteria>
- `audit.js` parses URL from `process.argv[2]`.
- Inserts row into `audits` table.
- Runs all 4 engine functions in the correct sequence (Playwright+Axe concurrently on same page, then Lighthouse, then Linkinator).
- Calls `normalizeFindings` and inserts into `findings` table.
- Writes a JSON file to `reports/`.
- Logs a summary to the console.
</acceptance_criteria>
</task>
