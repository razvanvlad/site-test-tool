# 02-02: Dashboard SPA

## Objective
Build the Vanilla JS Single Page Application (SPA) dashboard. Use OS-aware CSS variables for styling. Fetch data from the Express API, list audits, display a filterable table of findings with expandable rows, and provide inline UI for triaging (changing status, adding notes, viewing screenshot diffs).

## Tasks

```xml
<task>
  <action>Create `public/styles.css` with OS-aware CSS variables for dark/light mode (`--bg`, `--text`, `--border`, `--accent`). Style a simple layout with a sidebar (for audit list) and main content area (for findings table).</action>
  <read_first>
    - public/styles.css (if exists)
  </read_first>
  <acceptance_criteria>
    - public/styles.css contains `@media (prefers-color-scheme: dark)`
    - Variables like `--bg` and `--text` are defined
  </acceptance_criteria>
</task>

<task>
  <action>Create `public/index.html` referencing `styles.css` and `app.js`. Setup `<div id="sidebar">` for audits and `<div id="main">` for findings. Add `<select>` dropdowns for filtering findings by Severity and Category.</action>
  <read_first>
    - public/index.html (if exists)
  </read_first>
  <acceptance_criteria>
    - index.html contains `<link rel="stylesheet" href="styles.css">`
    - index.html contains `<script src="app.js"></script>`
  </acceptance_criteria>
</task>

<task>
  <action>Create `public/app.js`. On load, fetch `/api/audits` and populate the sidebar list. Clicking an audit should fetch `/api/findings/:auditId` and render a table in the main area. Implement filtering logic for the table based on the dropdown selects.</action>
  <read_first>
    - public/app.js (if exists)
  </read_first>
  <acceptance_criteria>
    - app.js contains `fetch('/api/audits')`
    - app.js contains `fetch('/api/findings/' + auditId)`
  </acceptance_criteria>
</task>

<task>
  <action>In `public/app.js`, build table rows that can expand when clicked to reveal finding details. Inside the expanded row, render inline inputs for `Notes`, a checkbox for `False Positive`, and a dropdown for `Status` (open/fixed/wontfix). Add event listeners that `PATCH /api/findings/:id` when these inputs change.</action>
  <read_first>
    - public/app.js
  </read_first>
  <acceptance_criteria>
    - app.js contains `fetch('/api/findings/' + id, { method: 'PATCH'`
  </acceptance_criteria>
</task>

<task>
  <action>In the expanded row, if `evidence_path` exists, display the "before" screenshot. Implement the "Click to toggle" UI: `<img src="evidence_path" onclick="cycleDiff(this, finding)" />`. The JS function should cycle the `src` between `evidence_path` (Before), `after_screenshot_path` (After, if exists), and `diff_image_path` (Diff, if exists).</action>
  <read_first>
    - public/app.js
  </read_first>
  <acceptance_criteria>
    - app.js contains logic to cycle image `src` between before/after/diff paths.
  </acceptance_criteria>
</task>
```
